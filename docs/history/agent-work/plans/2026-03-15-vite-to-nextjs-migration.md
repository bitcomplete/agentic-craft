# Vite to Next.js Migration Plan

> Status: Historical archived plan. Preserved from an earlier implementation session.
> Do not use this file as the current task list or repo workflow guide.
> Current onboarding starts at `AGENTS.md`, `docs/agent/README.md`, and `INTERACTIVE_SPEC.md`.

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate the Agentic Design System from Vite 7.2.4 SPA to Next.js 16 App Router with SSR, SEO, and React Server Components.

**Architecture:** Hybrid RSC -- layout/shell as server components, interactive page content as client components wrapped with `'use client'`. Each route gets a thin RSC `page.tsx` that exports metadata and renders the existing page content as a client component. Static generation (SSG) for all pages, deployed to Vercel.

**Tech Stack:** Next.js 16, React 19, TypeScript 5.9, Tailwind CSS 4 (PostCSS), shadcn base-nova, @base-ui/react, HugeIcons

**Spec:** `docs/history/agent-work/specs/2026-03-15-vite-to-nextjs-migration-design.md`

## Chunk 3: Page Migration

### Task 9: Rename and convert page files

**Files:**
- Rename + modify: all 8 page files from `src/pages/` to `src/views/`
- Delete: `src/pages/output.tsx`

**IMPORTANT:** Next.js reserves the `pages/` directory name for Pages Router. Having `src/pages/` alongside `app/` causes a build error: "pages and app directories should be under the same folder". The solution is to rename `src/pages/` to `src/views/`.

Each page needs:
1. Move from `src/pages/[name].tsx` to `src/views/[name]-content.tsx`
2. Add `'use client'` as the first line
3. Change default export to named export

- [ ] **Step 1: Rename directory and all page files**

```bash
git mv src/pages/observability.tsx src/views/observability-content.tsx
```

- [ ] **Step 2: Add 'use client' and change exports**

For each file, add `'use client'` as the very first line (before all imports), and change the default export to a named export.

Example for `demo-content.tsx` -- the first line becomes:
```typescript
'use client'
```

And the export changes from:
```typescript
export default function Demo() {
```
To:
```typescript
export function DemoContent() {
```

Apply this pattern to all 8 files:

| `trust-content.tsx` | `export default function Trust()` | `export function TrustContent()` |

# ToolCall Component Implementation Plan

> Status: Historical archived plan. Preserved from an earlier implementation session.
> Do not use this file as the current task list or repo workflow guide.
> Current onboarding starts at `AGENTS.md`, `docs/agent/README.md`, and `INTERACTIVE_SPEC.md`.

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create a universal `ToolCall` component with status states (running/completed/failed) that works standalone and inside ToolTree, replacing ToolTree's internal item primitives.

**Architecture:** ToolCall is the atomic unit for tool call visualization. It renders a borderless row with icon-mask, label, hover timestamp, and optional expandable content. ToolTree is refactored to wrap ToolCall children with its connector/spine visuals instead of having its own item primitives.

**Tech Stack:** React 19, TypeScript, Tailwind CSS v4, Hugeicons, shadcn registry

- [ ] **Step 2: Create tool-call.tsx**

Create `src/components/ui/tool-call.tsx` with:
- Internal context for expanded state (no external context dependency)
- `ToolCall` root: manages expanded state, renders icon with bg-mask, status-dependent styling
- `ToolCallLabel`: renders inside the trigger button
- `ToolCallContent`: expandable section, only renders when expanded
- Status logic:
  - `running`: icon gets `animate-tool-call-pulse`, no chevron
  - `completed`: static icon, chevron if ToolCallContent present
  - `failed`: `text-destructive` on label, auto-expands to show error + retry
- Detect ToolCallContent presence via `React.Children` scan for `data-slot="tool-call-content"`
- Icon rendering reuses the same pattern as ToolTree: `size-5` container, `size-6` bg-background mask, HugeiconsIcon inside
- Timestamp via hover state (same pattern as ToolTree's TreeTimestamp)
- Chevron: `ArrowRight01Icon`, rotates 90deg when expanded
- Error state: when `error` prop is set and expanded, render error text + optional retry button below content

Exports: `ToolCall`, `ToolCallLabel`, `ToolCallContent`

### Task 2: Refactor ToolTree to use ToolCall

Delete these components and their context (`ToolTreeItemContext`, `useToolTreeItem`). Remove them from exports.

- [ ] **Step 2: Update ToolTreeContent to wrap ToolCall children**

`ToolTreeContent` uses `React.Children.map` to wrap each child in a connector div:

```tsx
{items.map((child, i) => {
  const isLast = i === items.length - 1
  return (
    <div key={i} className="relative min-w-0">
      {isLast && (
        <div className="absolute w-px bg-background"
          style={{ top: 0, bottom: -24, left: -19 }} />
      )}
      <div className="absolute rounded-bl-lg border-l border-b"
        style={{ top: -5, left: -19, width: 30, height: 16, borderColor: "var(--tool-tree-connector)" }} />
      {child}
    </div>
  )
})}
```

This moves the connector rendering from inside ToolTreeItem (which no longer exists) into ToolTreeContent's wrapper.
